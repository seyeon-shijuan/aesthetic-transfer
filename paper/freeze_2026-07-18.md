## 1. Readability threshold τ

**τ = 0.30**

For attribute `a` and backbone `φ`, the gate is

```
gate_a(φ) = SRCC( g_a(φ(D_A_test)), a_gt )
```

where `g_a` is the linear probe fit on `D_A_train` and `D_A_test` is AADB TestNew
(n = 1,000).

**Decision rule.** Attribute `a` is retained only if `gate_a(φ) ≥ 0.30` for **all three**
P-gen backbones. If any backbone fails, `a` is classified **Unreadable** and excluded
from all `r_a`, `T_a`, and taxonomy analysis.

**Rationale for the all-three rule.** The `T_a` rank agreement statistic (Kendall's W)
is computed across backbones. If the surviving attribute set differed per backbone,
W would be computed over an unstable support and would not be interpretable. Requiring
unanimity fixes a common support at the cost of a smaller attribute set. This is the
conservative choice: it can only shrink the set of attributes we make claims about.

**Rationale for the value 0.30.** SRCC below 0.30 is conventionally described as weak
association. Setting τ higher (e.g. 0.50) risks emptying the taxonomy and leaving the
headline table with too few rows to support a claim. Setting τ lower (e.g. 0.10) risks
asserting that we "read" an attribute when the probe is tracking noise. Several AADB
attributes (notably motion blur and symmetry) are known to be hard to regress even for
supervised models, so a threshold that assumes all eleven are readable would be naive.

**Reporting obligation.** Unreadable attributes are reported explicitly in the taxonomy
table with their gate values, not silently dropped. An attribute that fails the gate is
a finding about the representation, not an inconvenience.

## 2. Feature definition

**Feature = `pooler_output` of each backbone. 768-d for all three.**

Verified by loading test: DINOv2 (1, 768), CLIP (1, 768), SigLIP2 (1, 768).

**Explicit prohibition: do not use `last_hidden_state[:, 0]`.**
SigLIP2 has no CLS token; it uses an attention-pooling (MAP) head. Index 0 of its
hidden states is the first *patch*, not a summary token. Using it would silently
degrade only SigLIP2 and would masquerade as a backbone-family difference.

**Preprocessing.** Each backbone uses its own native `AutoImageProcessor` configuration.
The same processor is applied to all three domains (AADB, AVA, BAID) without exception.

We do **not** force a single pixel-level transform across backbones. The requirement
that the comparison be controlled means identical pipeline code, identical images,
identical splits, identical head, identical seeds — not identical resize arithmetic.
Forcing a non-native transform onto a backbone would degrade that backbone specifically
and would be misread as a property of its representation.

**The constraint that does bind:** whatever transform a backbone uses, it must be
applied identically to all three domains. This is why the aspect-distorted `warp256`
release of AADB was rejected in favour of the original-resolution release. Had AADB
alone been pre-distorted, `shift_within = r_a(D_A) − r_a(D_P)` would have measured a
preprocessing artifact rather than a medium effect, and the attributes most sensitive
to that artifact (rule of thirds, symmetry, balancing elements, depth of field) are
precisely the ones expected to be informative.

## 3. Model lineup

| Role | Model | Identifier |
|---|---|---|
| P-gen-1 | DINOv2 ViT-B/14 | `facebook/dinov2-base` |
| P-gen-2 | CLIP ViT-B/16 | `openai/clip-vit-base-patch16` |
| P-gen-3 | SigLIP2 ViT-B/16 | `google/siglip2-base-patch16-224` |
| R-gen-1..3 | same three backbones, head re-fit on BAID_train | (cache reused) |
| P-spec-1 | Aesthetic Predictor V2.5 | `aesthetic-predictor-v2-5` (pip) + `google/siglip-so400m-patch14-384` |
| P-spec-2 | Q-Align / OneAlign | `q-future/one-align` |
| M-spec-1 | ArtiMuse | `Thunderbolt215215/ArtiMuse` |
| R-spec-1 | SAAN | BAID repo checkpoint (`model_best.pth`) |

**Scale is controlled at base for P-gen.** CLIP is more commonly used at ViT-L/14 in the
aesthetics literature. It is deliberately used at ViT-B/16 here so that the three P-gen
backbones differ in family and not in parameter count. Otherwise a difference attributed
to "backbone family" could be a difference in capacity.

**ArtiMuse variant.** The base checkpoint trained on ArtiMuse-10K is used. The
`ArtiMuse_TAD66K` and any AVA-finetuned variants are **excluded**. ArtiMuse's claim to
the multi-domain cell of the design rests on ArtiMuse-10K spanning graphic design, 3D,
AIGC, photography, and painting/calligraphy. A photography-finetuned variant would be
photo-specialized and would not occupy the cell it is being used to fill.

**Backbones are frozen.** Only a linear head is fit. No fine-tuning of any backbone,
no MLP heads. A linear probe read means the axis is present in the representation;
a nonlinear probe read would mean the probe constructed it.

**Additions and substitutions are closed as of this timestamp.** The only permitted
removal is failure of a documented verification step, which must be recorded in the
exclusion log with the failing command and error.

## 4. Data: measured, not assumed

### AADB (D_A)

Column order of `attMat.mat`, established by exhaustive numerical matching of every
column against every per-attribute list file (each column matched exactly one attribute):

```
col 0  = score
col 1  = BalacingElements   [sic — original spelling preserved]
col 2  = ColorHarmony
col 3  = Content
col 4  = DoF
col 5  = Light
col 6  = MotionBlur
col 7  = Object
col 8  = Repetition
col 9  = RuleOfThirds
col 10 = Symmetry
col 11 = VividColor
```

Cell-to-split correspondence: `cell0 = Train`, `cell1 = TestNew`, `cell2 = Validation`.

| Split | Measured n | Source |
|---|---|---|
| train | **8,458** | `imgListTrainRegression_*` |
| val | **500** | `imgListValidationRegression_*` |
| test | **1,000** | `imgListTestNewRegression_*` |

The commonly cited 8,500 for AADB train is a rounding of the released list, which
contains 8,458 entries. The measured value governs.

**AADB test is `TestNew`, not `Test`.** The distribution contains a second 1,000-image
list named `Test`. `attMat.mat` cell1 corresponds to `TestNew`. All gate computation and
all three-point application use `TestNew`. The `Test` list is unused.

**Filename normalization.** Original-resolution AADB test images carry a score prefix
(`0.500_name.jpg`) absent from the label lists. Join key is the basename after stripping
`^\d\.\d{3}_`.

### BAID (D_R)

| Split | Official | Measured | Excluded |
|---|---|---|---|
| train | 50,737 | **50,695** | 42 |
| val | 3,200 | **3,199** | 1 |
| test | 6,400 | **6,394** | 6 |

49 images (0.08%) are no longer served by the original host and could not be retrieved.
Zero corrupt files. Excluded images are enumerated in `baid_dropped.csv` and the filtered
splits are committed alongside the originals (`*_set_orig.csv`). Test-split loss is 0.09%.

**Validation split use is fixed now: ridge α selection only.** The AADB validation split
(500) serves the same purpose. Neither test split is ever used to select a hyperparameter.
Fixing this before any `R²` is computed removes the temptation to tune α on test when the
coverage denominator disappoints.

## 5. Label ceiling

> SAAN, an art-specialized model trained on BAID_train, achieves SRCC 0.472 on
> BAID_test — its own home turf.

Photo-aesthetics models on AVA routinely reach SRCC 0.70–0.75. The achievable ceiling on
BAID is therefore structurally lower, for reasons independent of any model under test.

**Consequence, declared in advance:** any model whose in-domain is AVA receives roughly
0.23 of "gap" for free, before medium transfer is considered at all. Raw `Δ` is therefore
**not** a headline quantity in this work. `G_P`, `G_R`, and coverage are. This ordering
was fixed before any `Δ` was computed and is not to be revisited after seeing them.

External corroboration: arXiv 2603.18108 reports that BAID labels derive from monthly
themed contest voting and reflect thematic fit and popularity rather than aesthetic
judgment per se, and that all evaluated methods reach only modest performance on it.

## 6. Execution environment

Two independent machines are provisioned identically as redundancy. SAAN was run on both;
results agreed to within 1e-4 (SRCC 0.4719871 vs 0.4719689), attributable to GPU
architecture, cuDNN kernel selection, and torch version differences.

**All final feature extraction and all final scoring run on a single machine.** Mixed
provenance across models in one table invites a question with no scientific content.
Cross-machine duplicates, where run, are recorded in the appendix as reproducibility
evidence only.

## 7. Reproduction notes (for appendix)

The official BAID `test.py` instantiates `BBDataset(type='validation')`, which loads
`val_set.csv` (3,200 rows), and then compares predictions against `test_set.csv`
(6,400 rows). It raises on execution. Evaluation here used `type='test'`. With that
correction the published numbers reproduce exactly (0.472 vs 0.473 SRCC,
0.467 vs 0.467 PLCC).
